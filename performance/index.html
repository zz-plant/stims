Stims guide

Compatibility and performance

Stims is built to start fast with demo audio, then scale up or down depending on your browser and hardware. This page sets expectations before the first click.

Best case

Recent laptop or desktop browser

You should get the fullest visual detail here, especially on newer hardware with stable graphics drivers.

Expected fallback

Lighter mode is still the normal product

If Stims drops into a lighter rendering path, the goal is smooth motion and responsive controls, not a broken session.

Fastest first run

Start with demo audio

Demo audio avoids permission friction and gets you to moving visuals immediately. Microphone and tab capture are optional upgrades.

What to expect

Browser and device guidance

Setup Expectation
Newer desktop browser Best detail, strongest chance of the full visual path.
Older laptop or integrated graphics Stims may prefer a lighter path to keep motion stable.
Phone or tablet Expect a cleaner, lighter session with touch-friendly controls.
TV-style browser Use the default quick looks first and keep extra tuning minimal.

When visuals look lighter

What fallback really means

  • Stims may reduce detail to keep motion smooth.
  • Preset behavior should still feel expressive and reactive.
  • You can keep using the app without digging through technical settings.

Quick fixes

If the first run feels rough

  • Use demo audio first instead of microphone or tab capture.
  • Open Look and try Battery saver or TV balanced.
  • Enable the steadier-hardware toggle inside Advanced tuning.

First-run flow

The simplest path

  1. Open Stims and press Play with demo audio.
  2. Switch presets until you find a look you want to stay with.
  3. Only then move to your own mic, room, tab, or YouTube audio.
