# Status — May 2026

Sprint 10 finalization. Current state of the Stims visualizer, as of 2026-05-16.

## What Changed (Sprints 1–10)

**Sprint 1:** Bootstrap. Certification corpus defined (23 presets). 4 bundled presets captured for visual measurement baseline.

**Sprint 2:** Compiler/test harness. `parity:test:corpus` and `parity:capture:corpus` CLI commands wired. Tolerance profiles established.

**Sprint 3:** Shader support inventory. Full audit of compiler AST and heuristic paths. 50 supported patterns cataloged. 10 unsupported/partial patterns and 6 silent fallback locations identified. Priority-ordered fix list generated (`docs/architecture/shader-support-inventory.md`).

**Sprint 4:** Module loading and toy runtime hardening. Two new fix categories emerged (11.2% and 8.8% of all fix commits). New review skills recommended for module-loading and toy-runtime.

**Sprint 5:** Rasterization fidelity audit. 7 divergence points between WebGL and WebGPU backends identified. Top 3: line thickness (2–10× difference), shape outlines (3–5× at small radii), textured shape fallback. Priority-ranked fix plan produced (`docs/architecture/rasterization-fidelity-audit.md`).

**Sprint 6:** Renderer setup hardening via fallback state machine implementation. Replaced 12-file implicit fallback chain with explicit FSM. 6 unsafe preconditions documented and guarded.

**Sprint 7:** Fallback state machine design formalized. 28 transitions across 12 states mapped. Timeout cleanup fire-and-forget gaps identified. Audio-renderer coupling design produced. renderScale multi-hop typing plan drafted (`docs/architecture/fallback-state-machine.md`).

**Sprint 8:** Shader fixes applied. XOR/pow GLSL mapping fixed. `bassAtt`/`midAtt`/`trebleAtt` camelCase aliases added to GLSL uniform map. tex3D volume sampler classification corrected.

**Sprint 9:** Public claim audit. 14 overclaims found (4 critical, 4 high, 6 medium). All critical fixed in `index.html`. 2 of 4 high fixed in docs. Fidelity tier labeling wired for measured presets (`docs/evidence/public-claim-audit.md`).

**Sprint 10:** Recurring fix patterns audit. 125 non-merge fix commits classified across 7 categories. 2 new instability categories identified (module loading 11.2%, toy runtime 8.8%). 20 hottest files ranked by fix density. Two new review skills recommended. CSS files found to have highest fix rates (40.9% for `base.css`). Fix-to-total commit ratio: 75%, suggesting tightening the quality gate (`docs/evidence/RECURRING_FIX_PATTERNS_AUDIT_2026-05.md`).

Highest-impact open gaps:

- **Shape thick-outline parity** (WebGPU 3–5× wider at small radii)

- **Tex3D on non-volume sampler** — possible garbage reads in GLSL path

## Next Target Classes (ordered by measured blockers)

### 1. Sampler-heavy presets — tex3D/volume handling

**Target preset:** `261-compshader-noisevol_lq` (projectM upstream fixture, unmeasured)

**Why first:** The tex3D/volume sampler fix was applied in Sprint 8, but this preset has zero evidence. A capture run would validate the fix end-to-end. This is the highest-confidence path to moving a preset from "unmeasured" to "baseline-measured."

**Effort:** Low — run capture pipeline, compare against projectM reference image in `visual-reference-manifest.json`.

### 2. Shader-heavy presets — remaining unsupported shader patterns

**Target preset:** `parity-shader-08` (semantic-only, no evidence)

**Why second:** 6 shader compiler gaps remain open. This preset, chosen for shader coverage, is the natural target for regression testing as each gap is closed. Currently parses but has zero visual evidence.

**Effort:** Medium — requires fixing at least 1–2 of the remaining 6 compiler gaps before capture.

### 3. Rasterization-heavy presets — line thickness and blend mode fixes

**Target presets:** `parity-feedback-orientation-01` (unmeasured), `parity-hybrid-09` (semantic-only)

**Why third:** Line thickness is the single highest visual-impact divergence. Fixing it would improve fidelity for ALL presets with waves, not just corpus targets. Blend mode gaps (subtractive, multiplicative) are secondary.

**Effort:** High — line thickness fix spans `wave-renderer.ts` and `renderer-adapter-webgpu-batching.ts`. Blend modes require new shader paths.

### 4. Capture pipeline run for 10 unmeasured presets

**Target:** All 10 unmeasured presets

**Why fourth:** These presets are in the certification corpus but were never run through the capture pipeline. Running them would surface parse errors, runtime crashes, and visual divergence immediately. 4 of 10 have projectM reference images waiting for comparison.

**Effort:** Low (infrastructure exists) — `bun run parity:capture:corpus -- --group projectm-upstream` followed by `bun run parity:capture:corpus -- --group local-custom-shape`.
